# 造一个真能用的 AI Agent

> 会聊天的 AI 很多，真能稳定做事的 Agent 很少。这本书讲后者。

很多人第一次做 Agent，都会经历同一个错觉：模型能回答问题，接上几个工具，好像就已经是 Agent 了。

直到它开始重复调用失败工具、把错误写进上下文、搜索十几次仍然停不下来、把错误记忆带进下一次会话、在上线后把 token 和时间一起烧掉。那一刻你会发现，Agent 的难点从来不只是“让模型调用工具”，而是让一个不稳定的智能系统在真实世界里可控、可验收、可上线。

这本书的核心判断很简单：

**Agent engineering 不是 prompt 技巧，而是一套围绕循环、工具、记忆、错误、安全和交付建立控制系统的工程。**

这是一本写给产品型技术读者、创业者、独立开发者和半技术背景实践者的中文工程书。你不必先成为职业程序员才能读懂它，但你需要愿意追问：一个 Agent 为什么会失控，系统应该在哪里设边界，什么结果才算真正完成。

它不是 AgentClaw 的产品手册，也不是英文文章区的翻译版。它来自真实的 agent engineering 现场：循环失控、工具误用、记忆污染、成本膨胀、浏览器感知、交付失败、安全边界和上线纪律。

AgentClaw 会作为贯穿全书的案例出现，因为这套框架确实经历过这些问题，也留下了可检查的源码和工程取舍。但这本书真正要讲的不是某个产品，而是一条从概念到生产系统的学习路径。

源码开源在 [github.com/vorojar/AgentClaw](https://github.com/vorojar/AgentClaw)。你可以把这本书当成路线图，把代码仓库当成现场。

## 读者承诺

读完这本书，你不只是知道 Agent 是什么，而是能理解一个真实 Agent 框架从 0 到 1 的关键决策：

- 循环怎么控制，才不会让 Agent 一直跑下去。
- 工具怎么设计，才不会把错误自由交给模型。
- 记忆怎么保存、更新和遗忘，才不会把“记住”变成污染。
- 成本怎么压低，才不会让后台调用悄悄吞掉预算。
- 错误怎么处理，才不会靠模型自觉来保证系统可靠。
- 安全怎么兜住，才敢让 Agent 接触 shell、文件、浏览器和真实用户。
- 最终怎么上线，才从“本地能跑”变成“别人能用”。

如果你已经会写代码，这本书会帮你少踩 Agent 工程里最昂贵的坑。如果你不是职业程序员，它会帮你看懂：AI 编程时代，一个人如何靠清晰目标、架构判断和严格验收，推动一个真实系统长出来。

这本书不承诺让你一夜之间拥有一个完美 Agent。它承诺的是更重要的东西：你会知道一个 Agent 系统什么时候是在变强，什么时候只是在变复杂。

## 和英文文章区的关系

英文 Agent Engineering publication 面向全球 agent 工程师，要求每篇文章都有一个可传播、可复用、可验证的工程论点。

中文书稿面向系统学习者，允许更长的叙事、更生活化的类比和更完整的学习坡度。两者共享 AgentClaw 的真实工程经验，但服务不同读者。
