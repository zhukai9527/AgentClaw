# System Guide

The guide section is for durable project knowledge: architecture, roadmap, and engineering lessons that remain useful after individual bug reports and task documents expire.

| Guide | Purpose |
|---|---|
| [Architecture](./architecture.md) | Package layout, data flow, module boundaries, database schema, and deployment shape. |
| [Roadmap](./roadmap.md) | Completed capabilities, current direction, and planned engineering work. |
| [Engineering Lessons](./engineering-lessons.md) | Design principles and production lessons collected while building AgentClaw. |

Short-lived implementation reports are deliberately excluded. If a document only proves that one repair happened once, it belongs in git history or a release note, not in the public guide.
